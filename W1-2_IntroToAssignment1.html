Assignment 1

Submission Deadline: 4th November 2020.
40% of assessed work (20% of module grade)

  • Read the paper by Sotak and Boyer. It can be downloaded from Moodle.
  • Write a well annotated ImageJ macro that implements the Laplacian of Gaussian (LoG) filter as described by either equation 1.1 or equations 1.2-1.4 of the paper. You can use the code “LoG.ijm” on Moodle as a starting point.
  • Write a well annotated ImageJ macro that implements a Difference of Gaussians (DoG) filter.
  • Write a README file that includes:
    • What each macro does
    • Instructions of how to use both macros
    • Instructions of how to produce an example output for each macro
    • The limitations of the methods and when/why you should use LoG or DoG.

Submission

  • Submit your README file named YourStudentID_README.txt and a .zip archive named YourStudentID_A1.zip via Moodle containing the following 4 (or 5) files:
    • README file
    • LoG macro
    • DoG macro
    • Resulting image after the LoG macro has been applied
    • Resulting image after the DoG macro has been applied
    • (Optional) Original example image, if you choose not to use one of the sample images in Fiji.
  • Note: You must submit your README file twice. Both inside the .zip file, for the peer-review, and once alone, for a plagiarism check.
  • You may help one another to implement the LoG and DoG methods as macros but must write your own comments and own README file.

Peer Review

Submission Deadline: 11th November 2020

You will be asked to peer-review the submissions of three other students. To do this, you will answer 5 yes/no questions about your colleague’s work. These are:
  • Does the README file state which version of Fiji/ImageJ should be used?
  • Can you reproduce the LoG example image?
  • Can you reproduce the DoG example image?
  • Can you apply the LoG macro to another image file?
  • Can you apply the DoG macro to another image file?


There will be comment boxes after each question, so you can expand on your answers if you want to. If you answer no to any of the above questions, you should state why here, e.g. “There was a typo in the macro submitted”. There is also an overall feedback box where you can leave further comments.
